## 🚀 v0.4.19 — 当前最新版本

**发布日期**: 2026-06-07

### 🔍 第三轮深度审查与修复
在 v0.4.18 基础上进行更全面的代码审查，覆盖数据持久化、状态管理、UI/UX、SFX 和算法边界情况，修复了 12+ 项问题。

#### 💾 数据持久化修复（严重）
1. **`dailyHistory` 未保存** — 修复了连续打卡历史在页面刷新后完全丢失的问题；`encodeStats` 现在正确写入 `dailyHistory` 字段。
2. **`skippedProblems` 未保存** — 修复了错题本在页面刷新后完全丢失的问题；新增 `skippedProblems` 的编码/解码器，并在 `decodeStats` 中限制最多保留 20 条。
3. **`keypadEnabled` 未保存** — 修复了虚拟键盘显示/隐藏偏好在刷新后恢复默认值的问题；`ToggleKeypad` 现在会调用 `saveCmd`。

#### 🧹 状态管理修复
4. **切换模式/难度时评分面板残留** — `SetGameMode`、`ChangeDifficulty`、`StartTimeAttack`、`StartReview`、`NewGame` 的所有分支现在都会清除 `lastRating`、`showSteps` 和 `stepByStep`，避免旧状态在新模式下继续显示。
5. **自定义模式答对后步骤面板残留** — Custom `handleCorrect` 现在会重置 `showSteps` 和 `stepByStep`；Custom `Skip` 也会清理 `lastRating` 和步骤状态。
6. **Hard 模式除法验证逻辑漏洞** — 修复了 `SubmitAnswer` 中 `parseExpr` 返回 `Nothing` 时意外调用 `handleCorrect` 的边界情况，现在会正确显示错误信息。
7. **评分面板动画不重置** — `Html.Keyed` 的 key 从基于评分内容改为基于 `totalGames + stars`，确保连续获得相同星级时动画仍能重新播放。
8. **`normalizeStarCounts` 负值防护** — 当传入列表长度超过 5 时，避免 `List.repeat` 接收负参数。
9. **"步步为营" 成就重复解锁** — 修复了首次答对且恰好 3 步时，`checkAchievements` 和 `bubuAchievement` 同时添加导致列表中出现重复项的 bug。

#### 🎮 UI/UX / SFX 修复
10. **虚拟键盘禁用状态** — 当输入框被禁用（如 `pendingNewCards` 或 TimeAttack 时间到）时，虚拟键盘的所有按钮（数字、运算符、退格、清除、提交）现在也会同步禁用，避免用户误触。
11. **成就解锁消息覆盖得分信息** — `buildMessage` 重构为在原有前缀消息后附加成就信息，TimeAttack 模式下用户现在能同时看到"+N分！+10秒！"和"🏆 解锁成就！"。

### 🔧 维护
- 全站版本号统一升级为 **v0.4.19**
- Service Worker 缓存版本从 `v21` 升级到 `v22`

## 🚀 v0.4.18

**发布日期**: 2026-06-07

### 🧹 第二轮深度修复与打磨
在 v0.4.17 基础上继续对评分系统、数据持久化、自定义/复习模式兼容性、UI/UX、SFX 和动画进行全方位深度审查，修复了 10+ 项细节问题。

#### ⭐ 评分系统完善
1. **评分面板可关闭 + 动画重置** — 新增右上角关闭按钮和 `DismissRating` 消息；使用 `Html.Keyed` 让每次新评分都重新播放 `popUp` 入场动画，连续答对时反馈更鲜明。
2. **五星音效去重** — 当 5 星解法同时解锁新成就时，不再重复播放两个 achievement 音效，避免刺耳叠加。
3. **评分详情兜底优化** — `solveTime == 0` 时不再显示"0秒内速解"，避免尴尬文案。

#### 🐛 数据与模式兼容性修复
4. **`dailyBestTime` 跨天污染** — 修复了昨日最佳时间会被继承到今天的问题：当存储的 `dailyCompletedDate` 与今天不符时，`dailyBestTime` 会正确重置为 0。
5. **导入数据后立即持久化** — `StorageLoaded` 处理完后会调用 `saveCmd`，确保通过「导入数据」功能恢复的数据刷新后不会丢失。
6. **分享后立即持久化** — `ShareProblem` 增加 `saveCmd`，`sharedCount` 递增后立即保存。
7. **复习模式 + Hard 难度冲突** — 修复了 Hard 难度下进入复习模式时，若错题不含除法解，系统会丢弃该题并重新发随机牌的 bug；复习模式现在独立运行，不受 Hard 规则过滤影响。
8. **复习模式清空后自动切换** — 当错题本在复习过程中被清空后，系统会自动切换回经典模式并生成新牌，不再以复习名义继续生成随机题目。
9. **新牌重置步骤面板** — `NewCards` 现在会同时重置 `showSteps` 和 `stepByStep`，避免换题后仍显示旧题的步骤。
10. **自定义模式答对后清理更多状态** — 自定义答对后也会重置 `showAllAnswers`、`inputHint` 和 `liveResult`，防止旧提示残留。

#### 🎨 UI/UX 改进
11. **面板层级修复** — `achievement-toast` 与 `combo-popup` 的 `z-index` 提高到 10003，确保不会被自定义面板（10002）遮挡。
12. **浅色主题对比度补完** — 为成就墙标题、已解锁成就徽章和连续打卡标题在浅色主题下统一使用深金色 `#c78d0a`，提升可读性。
13. **难度按钮移动端换行** — 为 `.difficulty-row` 添加 `flex-wrap: wrap`，避免 3 个难度按钮在极窄屏幕上溢出。
14. **减少动画偏好补完** — 为 `.card:hover` 添加 `transform: none`，尊重系统级减少动画设置。
15. **模式特定欢迎语** — `NewCards` 的 `baseMsg` 现在为 Custom 和 Review 模式提供专属文案，不再统一显示"新的一组牌！"。

### 🔧 维护
- 全站版本号统一升级为 **v0.4.18**
- Service Worker 缓存版本从 `v20` 升级到 `v21`

## 🚀 v0.4.17

**发布日期**: 2026-06-07

### 🔧 深度修复与体验打磨
对 v0.4.16 上线的解法评分系统进行全面审查，修复了算法、数据一致性、UI/UX、SFX 和自定义模式兼容性等方面的多项问题。

#### ⭐ 评分算法重构
- 修复了旧算法导致非 Hard 模式下难以获得 5 星的问题。
- 新算法从 **无除法奖励、括号对数、解题速度、难度加成** 四个维度评分，1-5 星分布更加合理。
- 自定义挑战不再继承 Hard 难度加成，确保公平性。
- 5 星标签改为「完美杰作」，与「解法大师」成就名称区分，避免混淆。

#### 🐛 Bug 修复
1. **评分数据一致性** — `totalRated` 现在只在成功生成评分时递增，避免计数与 `starCounts` 总和脱节。
2. **评分统计持久化防御** — 新增 `normalizeStarCounts`，从 localStorage 读取的星级分布会被规范化为固定 5 项长度，防止异常数据截断。
3. **自定义模式 + Hard 难度冲突** — 修复了玩家在 Hard 难度下进入自定义挑战时，若题目不含除法解，系统会错误地重新生成随机牌的 bug。
4. **信息栏文本溢出** — 为 `.message` 添加 `overflow-wrap: break-word;`，避免长评情信息在窄屏上撑破布局。

#### 🎨 UX 改进
5. **平均星级显示精度** — 统计面板中的平均星级现在固定显示一位小数（如 `4.5★`），避免整数平均值缺少小数位的歧义。
6. **评分面板自动清理** — 自定义模式下点击「自定义新题」时会清除上一局的评分展示。
7. **浅色主题可读性** — 为浅色主题下的评分标签添加深金色，确保在浅色背景上仍清晰可辨。
8. **移动端适配** — 为评分面板添加 `@media` 响应式样式，在 600px 以下屏幕自动缩小字号与内边距。

#### 🔊 SFX / 动画
9. **五星 celebration 音效** — 获得 5 星评价时会额外播放 achievement 音效，增强正反馈。
10. **消息栏简洁化** — 移除了消息栏中过长的评分详情，只保留星级与称号，详情留在评分面板中展示。
11. **减少动画偏好** — 确认 `.reduce-motion .rating-panel { animation: none; }` 已生效。

- 全站版本号统一升级为 **v0.4.17**

## 🚀 v0.4.16

**发布日期**: 2026-06-07

### ⭐ 新功能：解法评分系统
每次正确作答后，系统将根据以下维度为你的解法打出 1-5 星评价，并累计到个人评分统计中：
- **运算步数**：越少越好（3 步以内得满分）
- **括号复杂度**：越少越好
- **是否使用除法**：不用除法可获额外加分
- **解题速度**：5 秒内完成可获得速度加成
- **难度加成**：高级难度额外加分

五星解法将获得「完美杰作」称号！评分统计会持久化到 localStorage，可在统计面板查看平均星级和五星次数。

## 🚀 v0.4.15

**发布日期**: 2026-06-07

本次更新是对 v0.4.14 新上线的**自定义挑战模式**进行一次深度修复与体验打磨，同时修复了若干 UI/UX/SFX/无障碍层面的细节问题。

### 🐛 Bug 修复
1. **自定义挑战求解器缓存未更新** — `StartCustomChallenge` 验证题目可解性后会更新 `solverCache`，但此前成功进入挑战时未将新缓存写回模型，导致同一题目可能被重复计算。现已修复。
2. **切换游戏模式时自定义面板未关闭** — 从自定义面板打开状态切换到经典/每日/计时/复习模式时，面板会正确关闭，避免遮罩残留。
3. **自定义答对后面板焦点错误** — 答对后 800ms 自动重新打开自定义面板时，焦点现在正确落在自定义输入框 `custom-input` 上，而非底层主输入框。
4. **自定义答对后面板竞态修复** — 将 `handleCorrect` 中延迟打开面板的消息由 `ToggleCustomPanel` 改为专用的 `OpenCustomPanel`，避免用户在庆祝动画期间手动关闭面板后又被重新打开。

### 🎨 UX 改进
5. **自定义输入错误提示更精确** — 当输入包含非数字内容（如 `1,2,abc,4`）时，现在提示"需要恰好 4 个有效数字"，而非笼统的"需要恰好 4 个数字"。
6. **模式按钮栏移动端适配** — 为 `.mode-row` 添加 `flex-wrap: wrap`，5 个模式按钮在窄屏上会自动换行，避免溢出。
7. **自定义面板自动聚焦** — 打开自定义面板时，输入框会自动获得焦点，玩家可立即输入数字。

### 🔊 SFX / 无障碍
8. **自定义面板音效补齐** —
   - 打开/关闭自定义面板现在播放 `click` 音效
   - 关闭面板也会播放 `click` 音效
9. **减少动画偏好支持完善** — 为 `.tutorial-box`（自定义面板复用此样式）添加 `animation: none`，尊重系统级 `prefers-reduced-motion` 设置。

### 🔧 维护
- 全站版本号统一升级为 **v0.4.15**
- Service Worker 缓存版本从 `v17` 升级到 `v18`
